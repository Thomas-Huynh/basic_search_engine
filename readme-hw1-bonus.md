After figuring out how to compile the code with the gcov flag using make -f Makefile.coverage, I noticed that the HashTable code coverage was roughly 93% after first running it. I took a look into the gcov file and noticed that the FNV64Hash method was completely untested. I also saw that a branch in the LLIterator_Get method where the iterator was invalid. Then, I used the Test_HashTable class used in the test_hashtable.cc file and tried to copy the general style of the tests. 

I created 2 unit tests: one that tested FNV64Hash, and one that tested the aforementioned LLIterator_Get branch. To write the FNV64Hash test, I looked up the Wikipedia article on the FNV64Hash and was able to find a calculator that calculated the exact value, so I used a simple string ("hi") as my test case and compared method's output to the online calculator's. To create the LLIterator_Get test, I created a HashTable and filled it with simple key value pairs. Then, I iterated through all the HashTable's elements. Then, I attempted to go to the next element, which would trigger the !LLIterator_IsValid branch on LLIterator_Get.

Writing these 2 tests led my coverage percentage to go up to around 98%.
